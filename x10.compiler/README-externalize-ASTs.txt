AST externalization
-------------------

The X10 compiler, x10c, supports outputting the compiler's internal
representation of a source file into an XML file.  These files may be
processed by other tools and then read back by x10c and compiled to the
target language as if read from an x10 source file.

x10c supports two modes of AST externalization:

* -EXTERNALIZE_ASTS=true

  When -EXTERNALIZE_ASTS=true is given on the command-line, after
  type-checking each file A.x10, x10c will output a file A.x10ml in the
  output directory containing an XML representation of A.x10's abstract
  syntax tree and types.  The output directory is the directory where
  class files and objects files are placed, usually the current
  directory or another directory specified with the -d option.

  The outputted xml files can then be processed by external tools.

  x10c can be invoked on .x10ml files as well as .x10 files.  x10c will
  not type-check x10ml source files; the type information in the files
  is assumed to be correct.

* -XML_PROCESSOR=<program>

  When -XML_PROCESSOR=<program> is given on the command-line, x10c will
  output x10ml files as above.  x10c will invoke the given program on
  each x10ml file, wait for the program to complete, then read back in
  the x10ml file from the same path, and continue compilation.

  The external program should take one parameter on the command-line,
  the path to the x10ml file.


X10ML Format
------------

The X10ML format is closely tied to x10c's abstract syntax tree and type
system implementation.  The file consists of a single XML document tree
with tag <X10Source>.  <X10Source> has two immediate children,
<AbstractSyntaxTree> and <TypeSystem>.

<AbsractSyntaxTree> contains an XML form of the internal abstract syntax
tree representation.  Tags correspond to AST implementation type names
and to field names.

<TypeSystem> contains a list of key-value pairs that map identifiers to
type system objects (types, methods, variables, constraints, etc).  The
key identifiers are used in <AbstractSyntaxTree> and <TypeSystem> as
references to an object in the map.

    class Fib {
        int fib(int n) {
